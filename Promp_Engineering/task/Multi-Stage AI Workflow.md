# Multi-Stage AI Workflow Across UX Types

## Scenario Briefing

Create a workflow that solves a problem by chaining outputs between at least two different AI UX types (e.g., Chat → IDE, IDE → CLI).

**Example:** A chat-based tool generates structured content → an IDE-based AI formats and integrates it into a codebase.

---

## Learning Objectives

- Understand how to design AI workflows that span different UX types
- Practice integrating outputs from one AI tool into another
- Develop tool-agnostic workflows adaptable to different providers

---

## Pre-requisites

- Basic familiarity with:
  - One chat-based AI tool
  - One IDE-based AI tool
  - One CLI-based AI tool
- Understanding of prompt chaining concepts

---

## Required Tools / Datasets

- Any chat-based AI platform (e.g., ChatGPT, Gemini Chat)
- Any IDE with AI integration (e.g., VS Code + GitHub Copilot)
- Any CLI-based AI interface (e.g., Claude Code CLI, Gemini CLI)
- Test data or content relevant to the chosen workflow

---

## Instructions

1. Select a problem that can be split into multiple stages
2. Choose two AI tools from different UX categories
3. Design a workflow where the output from one tool becomes input for another
4. Implement and test the workflow end-to-end
5. Document your workflow with:
   - A diagram
   - Clear step-by-step notes

---

## Expected Deliverables

- Working AI workflow that runs across two UX types
- Workflow diagram
- Written documentation

---

## Evaluation Criteria

| Criteria | Weight | Description |
|----------|--------|-------------|
| Functionality | 40% | Workflow runs end-to-end without breaking |
| Adaptability | 20% | Minimal dependency on specific tools/models |
| Documentation | 20% | Clear diagram and explanation of workflow steps |
| Efficiency | 20% | Workflow meaningfully reduces manual effort |
